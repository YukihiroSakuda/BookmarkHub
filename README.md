# BookmarkHub

モダンなUIと使いやすい機能を備えたブックマーク管理アプリケーション。

## 機能一覧

### ブックマーク管理
- ブックマークの追加・編集・削除
- タイトル、URL、タグの管理
- ピン留め機能
- リスト表示とグリッド表示の切り替え
- アクセス回数の記録
- 最終アクセス日時の記録

### タグ管理
- タグの追加・編集・削除
- タグ別表示
- タグによるフィルタリング

### 検索機能
- タイトル、URL、タグによる検索
- リアルタイム検索

### ソート機能
- アクセス回数によるソート
- タイトルによるソート
- 作成日時によるソート
- 昇順/降順の切り替え

### インポート/エクスポート
- HTMLファイル形式でのエクスポート
- エクスポートしたHTMLファイルのインポート
- 重複チェック機能

### UI/UX
- モダンなデザイン
- レスポンシブ対応
- アニメーション効果
- ダークモード
- ファビコンの自動取得

## 使用方法

### ブックマークの追加
1. 「Add Bookmark」ボタンをクリック
2. タイトルとURLを入力
3. タグを追加（オプション）
4. 「Add Bookmark」をクリックして保存

### ブックマークの編集
1. ブックマークの編集アイコンをクリック
2. 必要な情報を更新
3. 「Save Changes」をクリック

### ブックマークの削除
1. ブックマークの削除アイコンをクリック
2. 確認ダイアログで「OK」をクリック

### ピン留め
- ブックマークのピンアイコンをクリックしてピン留め/解除
- ピン留めされたブックマークは上部に表示

### タグの管理
- 既存のタグをクリックしてフィルタリング
- 新しいタグはブックマーク追加/編集時に作成可能

### 表示モードの切り替え
- ヘッダーの表示モードアイコンでリスト表示/グリッド表示を切り替え

### 検索
- ヘッダーの検索バーでブックマークを検索
- タイトルで検索可能

### ソート
- ヘッダーのソートコントロールで並び替え
- アクセス回数、タイトル、作成日時でソート可能
- 昇順/降順の切り替え可能

### インポート/エクスポート
- ブックマークのインポート/エクスポート機能でデータのバックアップが可能
- HTMLファイル形式でエクスポート
- エクスポートしたHTMLファイルをインポート可能

## 注意事項

### データの保存
- ブックマークデータはブラウザのローカルストレージに保存されます
- ブラウザのデータをクリアすると、保存されたブックマークも消去されます
- 異なるブラウザやデバイスではデータは共有されません

### バックアップ
- 定期的にブックマークをエクスポートしてバックアップすることをお勧めします
- エクスポートしたHTMLファイルは安全な場所に保管してください

### ファビコン
- ファビコンはGoogleのfaviconサービスを使用して自動取得されます
- 内部ネットワークのURLや特殊なドメインの場合はファビコンが表示されない場合があります

## 技術スタック

- **フレームワーク**: Next.js 14
- **言語**: TypeScript
- **スタイリング**: Tailwind CSS
- **アイコン**: Lucide Icons
- **状態管理**: React Hooks
- **ストレージ**: LocalStorage

## セットアップ手順

1. リポジトリのクローン
```bash
git clone https://github.com/YukihiroSakuda/BookmarkHub.git
cd BookmarkHub
```

2. 依存関係のインストール
```bash
npm install
```

3. 開発サーバーの起動
```bash
npm run dev
```

4. ブラウザで確認
```
http://localhost:3000
```

## 開発ガイドライン

### プロジェクト構造
```
src/
  ├── app/              # アプリケーションのルート
  ├── components/       # 共通コンポーネント
  ├── types/           # 型定義
  └── utils/           # ユーティリティ関数
```

### コンポーネント設計
- 各コンポーネントは単一責任の原則に従う
- Propsの型定義を必ず行う
- 再利用可能なコンポーネントは`components`ディレクトリに配置

### スタイリング
- Tailwind CSSを使用
- カスタムカラーは`tailwind.config.ts`で定義
- コンポーネント固有のスタイルは該当コンポーネント内で定義

### 状態管理
- ローカルステートは`useState`を使用
- グローバルステートは必要に応じて`Context`を使用
- データの永続化は`LocalStorage`を使用

### 命名規則
- コンポーネント: PascalCase
- 関数・変数: camelCase
- 定数: UPPER_SNAKE_CASE
- ファイル名: コンポーネントと同じ名前（PascalCase）

### コミットメッセージ
- feat: 新機能
- fix: バグ修正
- docs: ドキュメント
- style: スタイル
- refactor: リファクタリング
- test: テスト
- chore: その他
